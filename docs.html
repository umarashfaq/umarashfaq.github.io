Clarity is dead simple to use, and most likely you would not need to refer docs once you get a hang of it. Here are some of the key concepts:

Key Concepts

  • Lines as sequence of boxes

    Each line of text, as long as it has text, is rendered as a box. Multiple lines are rendered as sequence of boxes, connected by arrows. At this point, the diagram flows from top to bottom.

  • Hashtags for special behavior

    Hashtags (a.k.a. directives) can be dropped on any line for special behavior. For example, if you want to make round corners out of an entry box, simply drop #enter directive anywhere on the line. It does not matter where a directive is placed in a line. See "Directives" section below for list of available directives.

  • Indentation for groups

    Indentation is typically used to group a number of statements together. For example, see the usage of #group and #if directives.

  • Comments

    Clarity supports C style comments with double forward slashes. Anything in comments is ignored.
